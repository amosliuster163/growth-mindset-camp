# 第51期 - 时间阻塞：为什么你的时间总是不够用

> **核心模型**：时间阻塞（Time Blocking）
> **一句话**：不是时间不够，是没规划。

---

## 📖 名人故事

埃隆·马斯克，同时管理SpaceX、Tesla、Twitter。

他的时间管理方法：**时间阻塞**。

**把一天分成多个时间块，每个块只做一件事。**

例如：
- 9:00-11:00：SpaceX工程会议
- 11:00-12:00：Tesla设计评审
- 13:00-14:00：邮件和消息
- 14:00-18:00：深度工作

**不是 multitasking，而是 single-tasking。**

---

## 🧠 核心洞察

**什么是时间阻塞？**

把日历分成多个时间块，每个块预先分配任务。

**时间阻塞 vs 待办清单**：

| 待办清单 | 时间阻塞 |
|----------|----------|
| 列出一堆事 | 指定时间做 |
| 不知道什么时候做 | 知道什么时候做 |
| 容易拖延 | 不容易拖延 |

**待办清单告诉你"做什么"，时间阻塞告诉你"什么时候做"。**

---

## ⚠️ 常见陷阱

| 陷阱 | 表现 | 破解 |
|------|------|------|
| 排太满 | "每个分钟都排了" | 留20%缓冲时间 |
| 不灵活 | "计划不能改" | 允许调整 |
| 不坚持 | "排了但不执行" | 当成约会对待 |

## ✅ 突破方法

### 1. 基础时间阻塞

| 时间 | 任务 |
|------|------|
| 8:00-9:00 | 晨间例行 |
| 9:00-11:00 | 深度工作 |
| 11:00-12:00 | 会议 |
| 12:00-13:00 | 午餐 |
| 13:00-14:00 | 邮件和消息 |
| 14:00-17:00 | 深度工作 |
| 17:00-18:00 | 回顾和计划 |

### 2. 主题日

| 星期 | 主题 |
|------|------|
| 周一 | 产品 |
| 周二 | 工程 |
| 周三 | 会议 |
| 周四 | 设计 |
| 周五 | 学习和总结 |

### 3. 批量处理

| 任务 | 批量处理 |
|------|----------|
| 邮件 | 每天2个时间块 |
| 消息 | 每小时看一次 |
| 会议 | 集中在同一天 |
